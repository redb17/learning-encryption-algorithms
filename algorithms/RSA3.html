A combination of RSA and Diffie Hellman is used to get all the pros minus all the cons of both protocols. Static RSA keys are first used to set authentication between the two parties. Then, ephemeral Diffie Hellman keys are exchanged and messages are shared between the two parties.

All the Pros:

  1. Static RSA keys can be used as RSA key generation is slow.
  2. Authentication is achieved due to RSA.
  3. Forward Secrecy is achieved since a new key is generated for each session; due to ephemeral Diffie Hellman keys as generation of these keys is fast.
