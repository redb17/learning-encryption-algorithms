RSA is also a public key exchange protocol. Two keys (key pair): public key and private key are generated by each of the two parties. Public key is made public and private key is kept private by each of the two parties. These two keys have the property that a message encrypted by either one key can only be decrypted by the other key of the key pair, hence these are called assymetric keys. These key pairs are generated by some key generation algorithms and libraries like ssh-keygen.

The algorithm:
Suppose P1 and P2 want to establish a secured session between them.

  1. P1 generates a key pair: pub1, pvt1. Same way, P2 generates a key pair: pub2, pvt2. Both pub1 and pub2 are made public.
  2. P1 encrypts a message with his private key pvt1 and then with P2's public key pub2 and sends to P2.
  3. P2 decrypts the message with his private key pvt2 and then with P1's public key pub1.

Authentication is done with RSA.
P2 upon successfully decrypting the message with pub1 can be sure of that the message is sent by P1 and no one else, since only P1 can encrypt a message which is decrypted successfully by pub1.
